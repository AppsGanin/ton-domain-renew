# TON Domains — продление `.ton`

Статический фронтенд: подключаете TON-кошелёк, видите свои `.ton` домены со
сроками истечения и продлеваете их — по одному или сразу несколько. Бэкенд не
нужен, размещается на GitHub Pages.

**Live:** <https://appsganin.github.io/ton-domain-renew/>

## Возможности

- **Подключение кошелька** через [TON Connect](https://docs.ton.org/develop/dapps/ton-connect/overview)
  (Tonkeeper, MyTonWallet и др.).
- **Список доменов** с датой истечения и понятным статусом: активен / истекает /
  скоро истечёт / истёк.
- **Продление в один клик** — на год.
- **Массовое продление**: отмечаете несколько доменов галочками и продлеваете
  одной операцией. Лимит сообщений в транзакции берётся из самого кошелька
  (`SendTransaction.maxMessages`): v4 — до 4 за раз, W5 — до 255. Если выбрано
  больше, чем кошелёк берёт за раз, операция автоматически разбивается на
  несколько последовательных транзакций.
- **Testnet** — добавьте `?testnet=true` к адресу страницы (см. ниже).
- **Без комиссии сервиса** — см. [«Сколько стоит продление»](#сколько-стоит-продление).

## Как это работает

- **Список доменов.** Запрос к TonAPI
  `GET /v2/accounts/{address}/dns/expiring?period=3660`. Поскольку домен `.ton`
  истекает максимум через ~366 дней после продления, большой `period` возвращает
  сразу все домены кошелька вместе с `expiring_at`. Адреса контрактов TonAPI
  отдаёт в raw-формате (`0:…`), а TON Connect требует user-friendly (`EQ…`),
  поэтому они нормализуются через `@ton/core` на границе API.
- **Продление.** На контракт домена (NFT) отправляется сообщение
  `change_dns_record` (op `0x4eb1f0f9`) с `key = 0` и без значения — «пустое»
  изменение записи, единственный эффект которого — обновление `last_fill_up_time`
  в контракте, что продлевает домен ещё на год. (Как в референсной реализации
  Tonkeeper.)

> Дата истечения обновляется не мгновенно: после подтверждения транзакции
> подождите 1–2 минуты и нажмите «Обновить».

### Сколько стоит продление

Само продление on-chain тратит только **газ** (порядка нескольких тысячных TON).
К сообщению прикладывается `0.02 TON` (`RENEW_AMOUNT_NANO` в `src/config.ts`) —
это **не комиссия сервиса**: остаток после газа остаётся на контракте самого
домена (идёт на его storage-ренту). Значение можно изменить в конфиге.

## Сеть: mainnet / testnet

По умолчанию — **mainnet**. Чтобы работать с тестовой сетью, добавьте
`?testnet=true` к URL:

```
https://appsganin.github.io/ton-domain-renew/?testnet=true
```

В этом режиме запросы идут на `testnet.tonapi.io`, транзакция помечается
`CHAIN.TESTNET`, а в шапке появляется бейдж **Testnet**. Кошелёк при этом должен
быть переключён в testnet (например, Tonkeeper → Settings → Dev → Testnet),
иначе он отклонит транзакцию из-за несовпадения сети.

## Деплой на GitHub Pages

В репозитории есть workflow [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml),
который собирает проект и публикует его на Pages.

1. Запушьте в репозиторий (ветка `main`).
2. **Settings → Pages → Build and deployment → Source: GitHub Actions.**
3. После пуша сайт соберётся и опубликуется автоматически.

Сборка использует `base: './'` (относительные пути), поэтому работает на любом
адресе Pages — `user.github.io/<repo>/`, собственном домене или в под-пути — без
правок конфигурации.

### TON Connect манифест

[`public/tonconnect-manifest.json`](public/tonconnect-manifest.json) публикуется
рядом с приложением, а его URL вычисляется автоматически от текущего адреса
страницы. Поля `url` / `name` / `iconUrl` влияют на то, как приложение
отображается в кошельке — поправьте их под свой адрес. `iconUrl` должен вести на
**PNG** (многие кошельки не принимают SVG); в репозитории лежит готовый
[`public/icon.png`](public/icon.png).

## Конфигурация

Основные константы — в [`src/config.ts`](src/config.ts). Опциональные переменные
окружения (см. [`.env.example`](.env.example)); помните, что любое `VITE_*`
попадает в публичную статическую сборку:

| Переменная | Назначение |
| --- | --- |
| `VITE_TONAPI_KEY` | Ключ TonAPI (без него запросы ограничены по частоте). |
| `VITE_TONCONNECT_MANIFEST_URL` | Переопределить URL TON Connect манифеста. |
| `VITE_GITHUB_REPO_URL` | Ссылка на репозиторий в футере. |

## Стек и структура

React 18 + TypeScript + Vite, TON Connect UI, `@ton/core`. Структура —
feature-based: доменная логика отделена от переиспользуемого UI, стили — в
CSS-модулях рядом с компонентами.

```
src/
├─ main.tsx                  # точка входа: провайдер TON Connect
├─ App.tsx                   # композиция страницы
├─ config.ts                 # конфиг и проверенные on-chain константы
├─ features/domains/         # фича «домены»
│  ├─ api.ts                 #   запрос списка к TonAPI
│  ├─ renew.ts               #   сборка транзакции продления
│  ├─ types.ts               #   модель Domain
│  ├─ useDomains.ts          #   загрузка/обновление списка
│  ├─ useRenewDomain.ts      #   продление (батчи, уведомления)
│  ├─ index.ts               #   публичный API фичи
│  └─ components/            #   DomainList, DomainCard
├─ components/               # секции и слой представления
│  ├─ layout/                #   Header, Footer
│  ├─ ui/                    #   Button, Badge, TonLogo, GithubIcon
│  ├─ Hero.tsx               #   экран до подключения
│  ├─ Toolbar.tsx            #   адрес кошелька + «Обновить»
│  └─ Notice.tsx             #   уведомления
├─ lib/                      # чистые утилиты
│  ├─ address.ts             #   сокращение/нормализация адресов
│  ├─ chunk.ts               #   разбиение на батчи
│  ├─ format.ts              #   статус и срок истечения
│  ├─ plural.ts              #   русские склонения
│  └─ wallet.ts              #   лимит сообщений кошелька
└─ styles/                   # дизайн-токены и базовые стили
```

## Замечания

- Без ключа TonAPI запросы ограничены по частоте, но для личного использования
  этого достаточно.
- Зависимости `@ton/core` / `@tonconnect` используют Node-глобалы
  (`Buffer`/`global`/`process`), которые полифилятся в [`vite.config.ts`](vite.config.ts).
